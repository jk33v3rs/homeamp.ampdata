"""README for HomeAMP V2.0 migrations.

This directory contains Alembic database migration scripts for HomeAMP V2.0.

## Initial Setup

1. Ensure `.env` is configured with correct `DATABASE_URL`
2. Install dependencies: `pip install -e .`
3. Generate initial migration: `alembic revision --autogenerate -m "Initial schema"`
4. Review generated migration in `migrations/versions/`
5. Apply migration: `alembic upgrade head`

## Common Commands

- **Generate migration**: `alembic revision --autogenerate -m "Description"`
- **Apply migrations**: `alembic upgrade head`
- **Revert last migration**: `alembic downgrade -1`
- **Show current version**: `alembic current`
- **Show migration history**: `alembic history`

## Migration Workflow

1. **Make model changes** in `src/data/models/`
2. **Generate migration**: `alembic revision --autogenerate -m "Description"`
3. **Review migration** in `migrations/versions/` (check for correctness)
4. **Test migration**: `alembic upgrade head` (dev environment)
5. **Test downgrade**: `alembic downgrade -1` (ensure reversibility)
6. **Re-upgrade**: `alembic upgrade head`
7. **Commit migration** to version control

## Best Practices

- Always review autogenerated migrations before applying
- Test both upgrade and downgrade paths
- Use descriptive migration messages
- Never edit applied migrations (create new migration instead)
- Backup database before applying migrations in production

## Migration Naming

Use clear, descriptive names:
- `alembic revision --autogenerate -m "add player session tracking"`
- `alembic revision --autogenerate -m "add indexes to config_values"`
- `alembic revision --autogenerate -m "add backup retention policy"`

## Troubleshooting

- **"Target database is not up to date"**: Run `alembic upgrade head`
- **"Can't locate revision"**: Check `alembic_version` table exists
- **Import errors**: Ensure `PYTHONPATH` includes project root
- **Connection errors**: Verify `DATABASE_URL` in `.env`
